# 系统管理弹窗统一迁移设计

## 目标

将 `/system/*` 范围内仍使用页面私有弹窗结构的界面迁移到共享 `MacDialog`，统一获得标题栏拖拽、最小化、最大化、左上角关闭、遮罩关闭和 Esc 关闭能力。

本次范围仅包含：

- 用户管理：用户新增/编辑弹窗、重置密码弹窗。
- 应用管理：应用新增/编辑弹窗。
- 数据字典：字典新增/编辑弹窗、字典项新增/编辑弹窗。

## 迁移方式

每个弹窗使用 `v-model` 绑定现有显示状态，并通过 `close-disabled` 复用现有提交或上传锁定状态。页面已有的关闭函数通过 `cancel` 事件继续执行，保证附属状态能够正确清理。

表单主体保留现有字段、校验、API 调用和提交函数。表单底部改用 `MacDialog` 的 `footer` 插槽，表单外提交按钮通过稳定的表单 `id` 关联原表单。

## 操作规则

- 移除页面私有遮罩、标题栏和右上角关闭按钮。
- 移除底部仅用于关闭弹窗的“取消”按钮。
- 保留“随机生成”“重置”等会改变表单内容的功能按钮。
- 保留保存、确认、确认重置等主提交按钮。
- 提交或上传期间继续禁止关闭、最小化和重复提交；最大化保持可用。
- 不改动删除确认、状态切换提示和页面级通知等非弹窗交互。

## 样式处理

删除仅服务于旧弹窗容器的遮罩、标题栏、关闭按钮样式。页面表单、预览卡片、字段网格和业务内容样式继续保留，通过 `panel-class` 提供页面级宽度与细节适配。

移动端沿用 `MacDialog` 现有底部弹窗布局，不增加页面私有媒体查询分支。

## 验证

- 增加系统弹窗组件测试，覆盖 3 个页面的 5 个弹窗均渲染为 `MacDialog`。
- 验证弹窗不存在右上角关闭按钮和底部取消按钮。
- 验证应用重置、密码随机生成等功能按钮仍存在。
- 验证提交锁定状态会传递给窗口控制和主操作。
- 运行组件测试、现有测试和生产构建。

## 范围边界

本次不迁移工作日志以外的业务应用弹窗，不修改后端接口，不统一页面表单视觉，也不重构现有业务状态管理。

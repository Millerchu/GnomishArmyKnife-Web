# 工作日志月视图设计

## 背景

当前工作日志页面以周视图为主，已经具备本周统计、7 天日历卡片、当天详情、年度列表和日志增删改能力。用户希望增加月视图，并已确认采用“最小增量日历”方向：月视图作为周视图的同级切换，同时保留独立的本月统计。

## 目标

- 在工作日志页面增加 `周视图 / 月视图` 切换。
- 月视图展示独立的本月统计，不复用本周统计标题和时间范围。
- 月视图以 7 列月历格子展示当月每日日志摘要。
- 单击日期选中当天，双击日期复用现有当天详情面板。
- 新增、修改、删除日志后刷新当前视图，并自动定位到日志所属日期。

## 非目标

- 不新增后端接口、数据库字段或表结构。
- 不改变现有工作日志 CRUD 协议。
- 不重做年度列表，也不把月视图扩展为复杂报表。

## 方案

前端在 `src/views/WorkLog.vue` 中增加视图状态 `calendarView`，取值为 `week` 或 `month`。周视图继续使用现有 `weekOffset`、`weekDays`、`weeklyLogs` 和 `weeklyStats`；月视图新增 `monthOffset`、`monthDays`、`monthlyLogs` 和 `monthlyStats`。

月视图数据复用现有 `listWorkLogs` 接口，按自然月构造 `startDate` 和 `endDate`。月历格子按周一作为一周第一天生成，包含当月前后的补位日期，但补位日期只做视觉占位，不参与本月统计。

页面顶部操作区根据当前视图切换导航按钮文案：周视图为上一周/下一周，月视图为上一月/下一月。新增、修改、删除仍共用现有表单和详情面板。

## 数据流

1. 页面初始化加载字典和当前周日志，默认进入周视图。
2. 用户切到月视图时，加载当前日期所在月份的日志。
3. `monthlyStats` 从 `monthlyLogs` 派生：参与项目、状态、地点、日志条数、有记录天数、人天总计、加班总计、周末日志数。
4. `monthSummaryMap` 按日期聚合月历格子的摘要：类型标签、项目文本、工作内容短文本、人天和加班。
5. 保存日志后通过日志日期判断所属周/月，刷新对应数据并选中日期。

## 组件与状态

- 列表状态：保留 `weeklyLogs`、`detailDayLogs`、`yearLogs`，新增 `monthlyLogs`。
- 视图状态：新增 `calendarView` 和 `monthOffset`。
- 派生状态：新增 `monthDays`、`monthlyStats`、`monthSummaryMap`，保持 `computed` 无副作用。
- API：继续通过 `src/api/workLog.js` 调用 `listWorkLogs`，页面中不拼接请求地址。

## 交互细节

- `周视图 / 月视图` 使用分段按钮，当前视图高亮。
- 月视图日期格显示日期、日志条数、类型摘要、项目摘要、工作内容摘要、人天和加班。
- 非当月补位日期降低透明度，不打开详情。
- 双击当月日期调用现有 `openDayDetail(date)`。
- 新增按钮默认使用当前选中日期；月视图选中日期时也遵循这一规则。

## 错误处理

月视图加载失败时沿用现有提示风格，提示“加载当月日志失败，请检查后端服务”。接口异常不清空已存在的周视图数据，避免用户在切换失败时丢失当前可见上下文。

## 测试

- 新增日期工具函数测试，覆盖月初、月末、跨年前后和周一作为一周第一天的补位逻辑。
- 新增月统计聚合测试，覆盖日志条数、有记录天数、人天、加班、周末日志数和去重项目/状态。
- 执行 `npm run test`。
- 执行 `npm run build`。

## 验收标准

- 默认仍进入现有周视图，原有新增、修改、删除、当天详情和年度列表行为可用。
- 切到月视图后能看到本月统计和完整月历。
- 月视图导航能切换上一月、下一月。
- 双击当月日期能打开当天详情。
- 日志保存或删除后，当前周/月视图数据刷新且日期定位正确。
